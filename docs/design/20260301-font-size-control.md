# 字体大小调整功能设计

## 需求
在 Markdown 阅读器中添加字体大小调整功能，方便用户根据阅读习惯和屏幕大小调整文字大小。

## 业界参考

### 1. **浏览器** - 快捷键 + 菜单
- **快捷键**：`Cmd/Ctrl + +` 放大，`Cmd/Ctrl + -` 缩小，`Cmd/Ctrl + 0` 重置
- **菜单**：查看 → 缩放
- **显示**：右上角显示当前缩放百分比（如 "110%"）
- **特点**：全局缩放，包括 UI 和内容

### 2. **VS Code** - 设置 + 快捷键
- **快捷键**：`Cmd/Ctrl + +/-` 调整编辑器字体
- **设置**：`editor.fontSize` 配置项
- **范围**：6px - 100px
- **默认**：14px（macOS），12px（Windows）
- **特点**：仅调整编辑器内容，UI 不变

### 3. **Notion** - 文本大小选项
- **位置**：右上角 "..." 菜单 → 文本大小
- **选项**：小、默认、大
- **特点**：预设档位，简单易用

### 4. **Medium** - 字体大小按钮
- **位置**：文章顶部工具栏
- **控件**：`Aa` 按钮，点击切换大小
- **选项**：小、中、大（3档）
- **特点**：直观的视觉控件

### 5. **GitHub** - 无字体调整
- 依赖浏览器缩放功能
- 代码块有独立的字体大小设置

### 6. **Typora** - 缩放功能
- **快捷键**：`Cmd/Ctrl + +/-/0`
- **菜单**：查看 → 缩放
- **范围**：50% - 200%
- **显示**：状态栏显示当前缩放比例
- **特点**：全局缩放，类似浏览器

## 设计方案

### 方案 A：浏览器风格 - 快捷键 + 百分比显示（推荐）

**控件位置：**
```
┌────────────────────────────────────────────────────────┐
│  文档 / README.md                                      │
│                                                        │
│            [⚙ 设置] [☁↑ 同步]  100%  2分钟前          │

```

**交互：**
- 快捷键：`Cmd/Ctrl + +` 放大，`Cmd/Ctrl + -` 缩小，`Cmd/Ctrl + 0` 重置
- 点击百分比：弹出下拉菜单，选择预设档位（75%, 100%, 125%, 150%, 200%）
- 显示：工具栏右侧显示当前缩放百分比

**特点：**
- ✅ 符合用户习惯（浏览器快捷键）
- ✅ 显示直观（百分比）
- ✅ 快速重置（Cmd/Ctrl + 0）
- ✅ 精细调整（快捷键每次 10%）

**实现：**
- 调整 `.markdown-body` 的 `font-size`
- 默认：16px (100%)
- 范围：12px (75%) - 32px (200%)
- 持久化到 localStorage

## 推荐方案：A（浏览器风格）

### 为什么选择方案 A？

1. **符合用户习惯** - `Cmd/Ctrl + +/-/0` 是最通用的缩放快捷键
2. **显示直观** - 百分比清晰表达缩放程度
3. **不占空间** - 百分比文字很小，不突兀
4. **灵活 + 快捷** - 既有快捷键，也可以点击选择预设

### 详细设计

#### 视觉设计
```
工具栏右侧：
[⚙ 设置] [☁↑ 同步]  100%  2分钟前
                     ↑
                  可点击

点击后弹出菜单：
┌──────────┐
│  75%     │
│  100% ✓  │
│  125%    │
│  150%    │
│  200%    │
└──────────┘
```

#### 交互细节

**快捷键：**
- `Cmd/Ctrl + +` 或 `Cmd/Ctrl + =`：放大 10%
- `Cmd/Ctrl + -`：缩小 10%
- `Cmd/Ctrl + 0`：重置为 100%

**点击百分比：**
- 显示下拉菜单
- 当前值有 ✓ 标记
- 点击选项立即应用

**范围限制：**
- 最小：75% (12px)
- 最大：200% (32px)
- 默认：100% (16px)

**显示样式：**
- 颜色：`#9ca3af`（和时间戳一样的浅灰色）
- 字体：等宽字体
- 可点击时 hover 变深灰色

#### 技术实现

**CSS：**
```css
.markdown-body {
  font-size: 16px; /* 默认 100% */
}

/* 通过 CSS 变量动态调整 */
:root {
  --font-scale: 1.0;
}

.markdown-body {
  font-size: calc(16px * var(--font-scale));
}
```

**JavaScript：**
```typescript
// 缩放状态
let fontScale = 1.0; // 100%

// 放大
function zoomIn() {
  fontScale = Math.min(fontScale + 0.1, 2.0);
  applyZoom();
}

// 缩小
function zoomOut() {
  fontScale = Math.max(fontScale - 0.1, 0.75);
  applyZoom();
}

// 重置
function zoomReset() {
  fontScale = 1.0;
  applyZoom();
}

// 应用缩放
function applyZoom() {
  document.documentElement.style.setProperty('--font-scale', fontScale);
  localStorage.setItem('fontScale', fontScale);
  updateZoomDisplay();
}
```

**持久化：**
- 保存到 `localStorage`
- 页面加载时恢复

#### 状态显示

```
75%   → 浅灰色
100%  → 浅灰色（默认）
125%  → 浅灰色
150%  → 浅灰色
200%  → 浅灰色

Hover 时变深灰色，表示可点击
```

## 其他考虑

### 是否影响 UI？
**建议：仅缩放内容区域**
- ✅ 缩放：`.markdown-body`（Markdown 内容）
- ❌ 不缩放：侧边栏、工具栏、面包屑

**理由：**
- UI 应该保持固定大小，保证可用性
- 用户主要想调整的是阅读内容的大小
- 类似 VS Code 的行为

### 是否需要缩放限制？
**建议：限制范围**
- 最小：75% (12px) - 太小会影响阅读
- 最大：200% (32px) - 太大会影响排版

### 是否需要平滑过渡？
**建议：不需要**
- 字体大小变化应该立即生效
- 过渡动画可能导致闪烁
- 浏览器缩放也是立即生效的
