# Case 13：非当前文件删除后的删除态、点击行为与刷新清理

## 目标
验证非当前文件被删除时：
- 立即进入删除态（红字 + 删除线 + `D`）；
- 点击该文件时不尝试刷新内容（显示缓存内容 + 删除提示）；
- 页面刷新后从打开列表中清理。

## 前置条件
- 简单模式。
- 已打开两个本地 Markdown 文件 A/B，且 B 为当前文件。

## 步骤
1. 删除非当前文件 A。
2. 观察 A 在侧边栏与 tab 的删除态。
3. 点击 A，确认内容区显示删除提示。
4. 刷新页面。

## 预期结果
- 步骤 1 后，A 显示 `D` 且文件名红字删除线。
- 步骤 3 后，不触发内容更新（保留缓存内容），并显示“已删除”提示条。
- 步骤 4 后，A 从列表和 tab 中移除。

## 失败信号
- A 删除后未进入删除态。
- 点击 A 后出现自动刷新替换或删除提示缺失。
- 刷新页面后 A 仍保留在打开列表。
