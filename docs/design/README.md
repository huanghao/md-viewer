# 设计目录说明

这个文件不再维护“全量文件清单”，避免和实际目录结构重复、且容易过时。

## 使用规则

1. 单文件设计：`docs/design/YYYYmmdd-*.md`
2. 多文件设计（设计文档 + 原型 + 调试记录 + 素材）：`docs/design/YYYYmmdd-<topic>/`
3. 新增多文件主题时，只需增加主题目录，不需要在本文件追加每个子文件列表。
