# Case 18：SSE 断线重连后继续接收 file-changed

## 目标
验证 SSE 连接断开并恢复后，当前文件仍能继续接收 `file-changed` 事件并进入可刷新状态。

## 前置条件
- 简单模式。
- 已打开本地 Markdown 文件并建立 SSE（`tests/e2e/runtime/case-18/e2e-sse-reconnect-a.md`）。

## 步骤
1. 打开测试文件 A。
2. 将浏览器网络切到离线，触发 SSE 断线。
3. 恢复在线并等待自动重连窗口。
4. 修改 A。

## 预期结果
- 重连后修改 A，出现 `M`/刷新按钮，手动刷新后内容更新为最新内容。

## 失败信号
- 网络恢复后修改文件，不再出现可刷新状态，或刷新后正文不更新。
