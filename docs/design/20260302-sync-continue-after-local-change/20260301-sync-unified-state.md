# 同步单页面联动设计（提案）

## 目标
- 不再出现“成功页/失败页/详情页”多页面切换。
- 始终停留在同一个同步面板，降低重试成本。
- 失败后默认保留输入并可直接重试。

## 单页面结构
- 上半区：输入与执行
  - 标题
  - 选择位置（最近父目录列表 + 手动输入 ID/URL）
  - 父文档元信息（输入失焦后获取标题并展示）
  - 命令预览
  - 同步按钮
- 下半区：联动状态区（同一块区域切换内容）
  - `idle`：不展示结果
  - `running`：展示执行中提示
  - `error`：展示失败信息 + 原始返回 + 直接重试
  - `success`：展示文档标题超链接 + 同步时间 + 原始返回

## 状态机
1. `idle -> running`：点击同步
2. `running -> error`：命令失败
3. `running -> success`：命令成功（未勾选“立即打开”）
4. `running -> idle`：命令成功且勾选“立即打开”（直接打开，不展示成功块）
5. `error -> running`：点击重试

## 与你要求对齐
- 失败后不关闭，留在当前页面，保留所有输入。
- 成功信息中的链接使用“文档标题（链接地址）”形态，不展示裸 URL 作为主要入口。
- 文本块支持局部选择；整体复制依旧有复制按钮。
