# Case 21：M 状态下禁止同步（需先刷新）

## 目标
验证当前文件在 `M` 状态时，同步按钮不可执行同步，用户必须先刷新。

## 前置条件
- 简单模式。
- 打开本地 Markdown 文件（`tests/e2e/runtime/case-21/e2e-sync-blocked-by-m.md`）。

## 步骤
1. 打开文件 A。
2. 修改 A，使其进入 `M`。
3. 观察同步按钮文案。
4. 点击同步按钮。

## 预期结果
- 同步按钮显示“先刷新后同步”。
- 点击后不会打开同步面板。

## 失败信号
- `M` 状态下仍可直接进入同步流程。
- 同步按钮文案未切换为阻断提示。
