## 文件更新提示 [进行中]

**开始时间：** 2026-02-28

**任务描述：**
在 UI 上提示文件的更新状态，包括本地文件更新、学城同步状态、文件不存在状态。

**设计文档：** [file-update-indicator.md](./design/file-update-indicator.md)

**当前状态：** 设计方案已完成（已补充文件不存在情况），等待用户确认

**设计原则：状态解耦，独立维护**

### 核心设计（已澄清）

现在有**四个独立的状态**：

**状态 1：isNew（未读）** - 已实现 ✅
- 含义：新加入的文件，从未被打开过
- 视觉：🔵 蓝色圆点
- 消失时机：点击文件切换时自动消失
- 字段：`isNew`

**状态 2：isDirty（已修改）** - 待实现 🔨
- 含义：本地文件已修改，需要**手动刷新**
- 判断方式：`lastModified > displayedModified`
- 视觉：待定（橙色圆点/文字标识/颜色变化）
- 消失时机：手动点击刷新按钮后消失
- 字段：通过时间戳对比判断，不需要新字段
- **关键特性**：这是 dirty 状态，不是点击就能消除的

**状态 3：isSynced（已同步）** - 待实现 🔨
- 含义：文件已同步到学城
- 判断方式：`syncedDocId` 是否存在
- 视觉：✓ 绿色勾（仅在工具栏显示）
- 位置：工具栏/同步按钮区域
- 字段：`syncedDocId`, `syncedUrl`, `syncedAt`

**状态 4：isMissing（文件不存在）** - 待实现 🔨
- 含义：本地文件被删除或移动
- 触发：SSE 推送 `file-deleted` 事件
- 视觉：待定（灰色+删除线+⚠️ / 灰色背景+⚠️ / 红色+⚠️）
- 优先级：最高（覆盖其他状态）
- 字段：`isMissing`
- 交互：提示用户，提供移除或查看学城链接选项

**四个状态完全独立，可以同时存在（isMissing 优先级最高）**

### 视觉布局（已明确）

1. **文件列表**：显示 isNew（🔵 蓝点）、isDirty（待定符号）、isMissing（⚠️），不显示学城同步状态
2. **工具栏/同步按钮区域**：显示学城同步状态（✓ 已同步）和刷新按钮（当 isDirty 时）

### 关键技术点

**如何检测文件状态变化？**
- 服务端通过 SSE 推送事件：
  - `file-changed`：文件内容变化
  - `file-deleted`：文件删除
  - `file-added`：文件新增（可能是恢复）
- 客户端只更新状态标记，不立即重新加载内容

**切换文件时的行为？**
- 点击文件切换：**不自动刷新**
- `isNew`（蓝点）消失（文件已被打开过）
- `isDirty`（待定符号）保留，需要手动刷新
- `isMissing`（⚠️）保留，提示用户文件不存在
- 工具栏显示相应的操作按钮

**手动刷新？**
- 点击工具栏的"刷新"按钮
- 或点击文件列表中的 dirty 标识（可选）
- 刷新后 `displayedModified = lastModified`，dirty 标识消失
- 可选：刷新时整体闪烁黄色，提示已刷新
