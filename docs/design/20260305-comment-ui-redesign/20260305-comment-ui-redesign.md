# 20260305 评论界面与交互重设计（原型说明）

文档定位：设计方案（待评审）

## 待决策项（先确认）

1. 评论面板默认宽度是否固定为 `360px`，还是支持用户拖拽记忆？
2. 历史评论是否默认折叠，仅在用户点击“历史”后加载？
3. 重定位失败（orphan）是否允许“强行绑定到最近段落”，还是只允许人工确认？
4. v1 是否包含评论图片上传（建议支持，先限制 3 张）？

## 目标

1. 评论交互不依赖模态框。
2. 文档缩放、滚动、编辑时，评论仍能稳定关联到原文。
3. 在文档变化后，评论能自动重定位，并显式暴露置信度与失败状态。

## 信息架构

1. 文档区：显示正文与锚点高亮。
2. 右侧评论面板：
- 顶部：筛选（全部/未解决/已解决/历史）+ 列表密度切换（默认/极简）
- 中部：评论线程列表
- 底部：新建评论输入区（内嵌）
3. 历史区：展示已解决/原文已删除评论，支持 reopen。

## 核心交互

1. 选中文本 -> 行内浮层按钮“评论” -> 右侧打开线程输入。
2. 点击评论卡片 -> 文档区滚动并高亮对应锚点。
3. 点击锚点 -> 评论面板定位到对应卡片。
4. 评论提交中若有上传任务未完成，禁止提交并提示。
5. 文档更新后自动重定位：
- `exact`：无感通过
- `reanchored`：轻提示“已自动重定位”
- `orphan`：卡片标红并提示“需确认定位”

## 状态语义（v1）

1. 线程状态：`open` / `resolved`
2. 锚点状态：`exact` / `reanchored` / `orphan`
3. 列表模式：`default` / `simple`
4. 视图分区：`active` / `history`
