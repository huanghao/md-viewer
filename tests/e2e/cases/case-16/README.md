# Case 16：当前文件连续快速修改后的最终收敛

## 目标
验证当前文件在短时间连续写入时，先进入可刷新状态（`M`/刷新按钮），手动刷新后正文收敛到最后一次内容。

## 前置条件
- 简单模式。
- 打开本地 Markdown 文件并保持为当前文件（`tests/e2e/runtime/case-16/e2e-rapid-write-a.md`）。

## 步骤
1. 在 case 目录内创建并打开测试文件 A。
2. 在短时间内连续多次改写 A（不同版本内容）。
3. 观察进入可刷新状态后，执行手动刷新。

## 预期结果
- 内容区在手动刷新后显示最后一次写入内容。
- 刷新后 `M` 状态消失。

## 失败信号
- 手动刷新后正文不是最后一次版本。
- 刷新后仍长期停留在中间版本或持续 `M`。
