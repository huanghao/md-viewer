# 文档模板

可用于新增 `docs/` 下文档时快速起草，降低结构不一致和重复内容风险。

```md
# <文档标题>

文档定位（<用户/设计/测试/证据/专题/归档>）：
- <这份文档负责什么>
- <这份文档不负责什么；权威文档链接>

## 背景

<为什么需要这份文档>

## 结论（先看）

1. <关键结论 1>
2. <关键结论 2>

## 正文

<按主题展开>

## 相关文档

- <链接 1>
- <链接 2>
```

使用建议：
1. 同一主题只保留 1 份权威文档，其他文档只做引用。
2. 非独一份文档（设计/调研/梳理/复盘）文件名使用 `YYYYmmdd-` 前缀。
3. 一次性调研/复盘完成后迁移到 `docs/archive/YYYY-MM/`。
4. 命令或配置变更时同步更新 `man/`、`docs/配置说明.md`、`showHelp()` 说明。
