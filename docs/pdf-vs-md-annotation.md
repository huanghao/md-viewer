# MD vs PDF 评论功能对比

## MD 已实现的功能
1. ✅ 划词选择 → 弹出评论框
2. ✅ 提交评论 → 高亮标记
3. ✅ 点击 mark → 显示评论详情 popover
4. ✅ 右侧列表点击 → 跳转到对应位置
5. ✅ 评论状态：已解决/未解决/孤儿评论
6. ✅ 评论回复（线程讨论）
7. ✅ 删除评论
8. ✅ 精确锚点（字符级偏移）

## PDF 当前实现
1. ✅ 划词选择 → 弹出评论框
2. ✅ 提交评论 → 高亮标记（整行）
3. ✅ 点击 mark → 显示评论详情 popover（刚实现）
4. ✅ 右侧列表点击 → 跳转到对应位置（刚实现）
5. ✅ 评论状态：已解决/未解决
6. ✅ 评论回复（线程讨论）
7. ✅ 删除评论
8. ⚠️ 锚点精度：item 级（整行），非字符级

## 主要差异
| 功能 | MD | PDF | 说明 |
|------|-----|-----|------|
| 选择精度 | 字符级 | 行级 | PDF items 是整行文本 |
| 锚点稳定性 | 高（字符偏移） | 高（item 索引） | 都稳定 |
| 视觉效果 | 精确高亮 | 整行高亮 | PDF 粒度粗 |
| 上下文 | 50字符 | 10 items | PDF 已增加 |
| 多页支持 | N/A | 支持 | PDF 特有 |

## 可能的改进
1. 精确子词选择：需要坐标→字符映射（复杂）
2. 更好的视觉高亮：给 mark 加背景色而非整行
