# Case 19：工作区轮询不应覆盖目录手动折叠状态

## 目标
验证在工作区模式下，用户手动折叠目录后，即使后台轮询触发重新扫描，也不会把该目录重置为展开。

## 前置条件
- 侧边栏模式为 `workspace`
- 有一个已展开工作区，指向仓库根目录
- 文件树中存在 `docs` 目录

## 步骤
1. 打开页面，等待工作区树加载完成。
2. 定位 `docs` 目录，确认初始为展开态（`▼`）。
3. 点击 `docs` 目录名称，折叠为 `▶`。
4. 等待超过轮询周期（> 1500ms）。

## 验收
- `docs` 目录仍保持折叠态（`▶`）。
- `docs` 的子树容器不应重新出现。
