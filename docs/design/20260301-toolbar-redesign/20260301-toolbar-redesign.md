# 工具栏右侧区域重新设计

## 现状问题

当前工具栏右侧包含：
- ⚙️ 设置按钮（emoji）
- 🔄 刷新按钮（蓝色边框，很突兀）
- 🔄 同步按钮（灰色边框）
- 最后修改时间（文字）

**问题：**
1. 风格老气，emoji 图标不够现代
2. 刷新按钮颜色突兀（蓝色边框高饱和度）
3. 各个元素之间没有统一的设计语言
4. 缺少层次感和呼吸感

## 设计参考

研究了以下现代产品的工具栏设计：

### 1. **Figma** - 极简图标 + 分组
- 所有操作用纯图标（SVG），无边框
- 相关功能用细分割线分组
- Hover 时显示浅灰背景
- 活跃状态用蓝色高亮

### 2. **Linear** - 幽灵按钮 + 精致间距
- 透明背景，极细边框（仅 hover 时显示）
- 图标和文字组合，间距精致
- 统一的 6px 圆角
- 柔和的灰色系

### 3. **Notion** - 无边框 + 分组容器
- 所有按钮无边框，纯图标
- 相关按钮放在同一个容器内
- Hover 时整个容器浅灰背景
- 极简、干净

### 4. **VS Code** - 纯图标 + 状态指示
- 工具栏全部用 SVG 图标
- 不同状态用颜色区分（灰色=正常，蓝色=活跃，橙色=警告）
- 无边框，hover 时显示背景
- 信息用小标签显示

### 5. **GitHub** - 图标 + 文字混合
- 主要操作：图标 + 文字
- 次要操作：仅图标
- 统一的灰色系
- 清晰的视觉层次

## 设计方案

### 方案 A：Figma 风格 - 纯图标 + 分组（推荐）

```
┌─────────────────────────────────────────────────────────────┐
│  文档 / 设计 / README.md              [⚙] │ [↻] [↑] • 2分钟前 │
└─────────────────────────────────────────────────────────────┘
```

**特点：**
- 所有按钮都是纯图标（SVG），无文字
- 用细分割线（`│`）分组
- 刷新图标用橙色圆点提示有更新
- 时间戳用小字体 + 圆点分隔
- 所有按钮 hover 时显示浅灰背景
- 统一的 24px × 24px 可点击区域

**图标设计：**
- ⚙ 设置：齿轮图标
- ↻ 刷新：圆形箭头（有更新时显示橙色圆点）
- ↑ 同步：向上箭头（已同步显示绿色对勾）

### 方案 B：Linear 风格 - 幽灵按钮

```
┌─────────────────────────────────────────────────────────────┐
│  文档 / 设计 / README.md         [⚙ 设置] [↻ 刷新] [↑ 同步] 2分钟前 │
└─────────────────────────────────────────────────────────────┘
```

**特点：**
- 图标 + 文字组合
- 透明背景，无边框
- Hover 时显示极细边框 + 浅灰背景
- 刷新按钮在有更新时显示
- 时间戳独立显示，不可点击

### 方案 C：VS Code 风格 - 状态色彩化

```
┌─────────────────────────────────────────────────────────────┐
│  文档 / 设计 / README.md              [⚙] [↻●] [↑✓] 2分钟前 │
└─────────────────────────────────────────────────────────────┘
```

**特点：**
- 纯图标，无文字
- 用颜色表达状态：
  - 刷新图标：灰色（无更新）/ 橙色圆点（有更新）
  - 同步图标：灰色（未同步）/ 绿色对勾（已同步）
- 最小化视觉噪音

### 方案 D：Notion 风格 - 分组容器

```
┌─────────────────────────────────────────────────────────────┐
│  文档 / 设计 / README.md         ┌──────────────┐  2分钟前   │
│                                   │ ⚙  ↻  ↑     │           │
│                                   └──────────────┘           │
└─────────────────────────────────────────────────────────────┘
```

**特点：**
- 操作按钮放在一个容器内
- 容器有浅灰背景
- Hover 时整个容器加深
- 时间戳独立在右侧

## 交互设计

### 刷新按钮
- **默认状态**：隐藏
- **有更新时**：显示，图标右上角橙色圆点
- **Hover**：浅灰背景，tooltip "文件已更新，点击刷新"
- **点击后**：圆点消失，内容刷新

### 同步按钮
- **未同步**：灰色图标
- **已同步**：绿色对勾
- **同步中**：旋转动画
- **Hover**：浅灰背景，tooltip 显示同步状态

### 设置按钮
- **默认**：灰色图标
- **Hover**：浅灰背景
- **点击**：打开设置面板

### 时间戳
- **格式**：相对时间（"2分钟前"）
- **颜色**：浅灰色 `#6b7280`
- **字体**：12px
- **不可点击**

## 推荐方案

**首选：方案 A（Figma 风格）**
- 最现代、最简洁
- 纯图标，视觉噪音最小
- 用颜色和状态表达信息
- 分组清晰

**实施优先级：**
1. P0：替换 emoji 为 SVG 图标
2. P0：统一按钮样式（无边框，hover 显示背景）
3. P1：添加状态指示器（橙色圆点、绿色对勾）
4. P1：优化间距和分组
5. P2：添加动画效果

## 技术实现要点

1. **SVG 图标组件化**：创建 `icons.ts` 统一管理
2. **CSS 变量**：定义统一的颜色、间距、圆角
3. **状态管理**：刷新按钮根据 `isDirty` 显示/隐藏
4. **Tooltip**：使用 `title` 属性或自定义 tooltip 组件
5. **响应式**：小屏幕时隐藏文字，仅显示图标
