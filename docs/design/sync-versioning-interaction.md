# 同步版本递增：交互设计（含步骤与恢复）

日期：2026-03-02
状态：Draft

## 1. 目标
- 本地文件修改后，用户可继续同步（每次新建文档，`-vN` 递增）。
- 处于 `M` 时禁止同步，明确提示“先刷新后同步”。
- 同步过程可见步骤状态，失败可重试/放弃/还原。
- 历史记录按时间倒序展示，不只显示最后一次。

## 2. 信息架构
- 主入口：工具栏同步按钮。
- 主操作面：右侧非模态“同步步骤面板”。
- 历史区：面板底部，按时间倒序。
- 危险操作：面板“更多”菜单中的“一键清除 / 一键还原”。

## 3. 按钮状态与触发
1. `UNLINKED`：`[☁↑ 首次同步]`
2. `READY`：`[↑ 继续同步(vN+1)]`
3. `BLOCKED_M`：`[先刷新后同步]`（禁用）
4. `SYNCING`：`[⏳ 同步中]`（禁用）
5. `FAILED`：`[! 同步失败，查看步骤]`

规则：
- `M` 优先级高于同步按钮可用性。
- 远端文件（URL）不显示同步按钮。

## 4. 步骤面板（固定 4 步）
1. 预检查：本地文件、非远端、非 `M`。
2. 计算版本：生成 `vN+1` 与目标标题。
3. 创建文档：调用 `km-cli doc create`。
4. 写本地状态：写入 active 版本与历史记录。

每步显示：
- 进行中 / 成功 / 失败
- 失败摘要 + 展开原始输出

## 5. 失败恢复策略
1. 重试当前步骤：仅重做当前失败步骤。
2. 重试整次同步：从步骤 1 重新执行。
3. 放弃本次：本次标记 `abandoned`，active 不变。
4. 一键还原：active 指针恢复到最近成功版本。
5. 一键清除：清空该文件本地同步状态，回到 `UNLINKED`。

## 6. 历史区设计（倒序）
- 排序：`syncedAt desc`。
- 每行字段：版本、标题、状态、时间、docId、链接。
- 状态颜色：
  - success：绿色
  - failed：红色
  - abandoned：灰色

## 7. 关键微交互
- 开始同步后自动滚动到“步骤区”。
- 当前步骤高亮，完成后折叠日志。
- 失败步骤保持展开，默认聚焦错误信息。
- 成功后不弹成功落地页，仅更新按钮与历史首行。

## 8. 验收检查单
1. 文件 `M` 时无法同步。
2. 每次继续同步创建新文档（`-vN`）。
3. 历史按时间倒序。
4. 失败不会覆盖当前 active 版本。
5. 一键清除 / 一键还原可用。
