# Case 11：当前文件被删除后的可见提示与删除态样式

## 目标
验证当前打开文件被删除后，用户能在不刷新页面的情况下清晰感知状态：
- 侧边栏：红字 + 删除线 + `D`
- Tab：红字 + 删除线
- 内容区：持续删除提示条（不是一次性 toast）

## 前置条件
- 以简单模式运行。
- 使用仓库内测试文件并通过 UI 打开。

## 步骤
1. 创建并打开 `docs/design/e2e-delete-visibility-check.md`。
2. 删除该文件（触发 `file-deleted` 事件）。

## 预期结果
- 当前文件在侧边栏显示删除态（红字、删除线、`D`）。
- 当前 tab 显示删除态（红字、删除线）。
- 内容区顶部显示持续删除提示条。

## 失败信号
- 删除后没有持续提示，只出现短暂 toast。
- 侧边栏或 tab 未进入删除态。
