# Case 10：file-changed 交互（当前/非当前都不自动刷新）

## 目标
验证文件变化事件在当前实现下的三条核心行为：
- 当前文件变化后不自动更新正文，显示 `M`，需手动刷新。
- 非当前文件变化后显示 `M` 状态（不自动刷新正文）。
- 切换到带 `M` 的文件后，正文保持旧内容，显示 diff/refresh 按钮，需手动刷新。

## 前置条件
- 启动本地服务并建立 SSE 连接。
- 使用本地临时 Markdown 文件 A/B。

## 步骤
1. 打开 A 和 B，保持 B 为当前文件。
2. 修改 B 内容。
3. 手动刷新 B。
4. 修改 A 内容。
5. 点击带 `M` 的 A。
6. 手动刷新 A。

## 预期结果
- 步骤 2 后，正文仍为旧内容，B 显示 `M`。
- 步骤 3 后，正文出现 B 的新内容，且 B 的 `M` 消失。
- 步骤 4 后，A 在侧边栏显示 `M`。
- 步骤 5 后，正文仍为 A 的旧内容，`M` 保留，diff/refresh 按钮出现。
- 步骤 6 后，正文出现 A 的新内容，且 A 的 `M` 消失。

## 失败信号
- 当前文件修改后自动刷新（违反当前设计）。
- 非当前文件修改后不显示 `M`。
- 切换到带 `M` 的文件后自动刷新（违反当前设计，会导致 diff 按钮不出现）。
- diff/refresh 按钮切换后不出现。
