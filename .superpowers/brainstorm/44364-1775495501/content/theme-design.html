实现架构

方案 A：CSS 变量主题切换（推荐）

📦
每套主题是一个 CSS 文件，通过 CSS 变量覆盖实现。切换主题只需替换一个 <link> 或动态注入 style，无需重新渲染 Markdown。
💾
主题偏好存 localStorage，重启后恢复。配置 key：md-viewer:theme（如 github / notion / bear）
🎨
代码高亮主题独立，可与 Markdown 主题组合。highlight.js 内置主题直接切换，Shiki 需要服务端渲染或 WASM。
⚙️
入口放在设置对话框，两个下拉：「正文主题」+「代码高亮」。也可在工具栏加快捷切换按钮。

方案 B：保持 highlight.js，只换 Markdown 主题（更轻量）

✅
不引入 Shiki（避免 WASM 依赖），只做 Markdown 样式多主题。代码高亮保持 highlight.js，但可以换主题（github / github-dark / atom-one-dark 等，都是纯 CSS）。
📉
实现成本低：新增 3-4 套 CSS 文件 + 设置里加主题选择器，不动渲染逻辑。
