文件状态视觉效果预览

VS Code 风格的文件状态标识方案

文件列表（侧边栏）
📄 README.md 正常状态
📄 new-doc.md 新文件（未读）
M 📄 guide.md 已修改
M 📄 tutorial.md 新文件 + 已修改（只显示 M）
D 📄 deleted.md 文件不存在
说明：
• 🔵 蓝点 = 新加入的文件，从未被打开过（已实现）
• M = Modified，本地文件已修改，需要手动刷新
• D = Deleted，文件不存在（被删除或移动）
• 字母使用等宽字体，加粗，颜色编码
• 优先级规则：D > M > 🔵（当多个状态同时存在时，只显示优先级最高的）
标签页（顶部）
README.md
guide.md
deleted.md
tutorial.md
说明：
• 删除的文件在标签页中显示为红色文字 + 删除线
• 与 VS Code 的标签页样式保持一致

VS Code 风格的优势：
• 用户熟悉度最高（VS Code 是最流行的编辑器）
• 字母含义清晰，无需学习（M=Modified, D=Deleted）
• 字母 + 颜色双重编码，识别度高
• 与现有蓝点（相当于 U=Untracked）风格统一
• 在标签页中配合红色删除线，视觉效果完整
