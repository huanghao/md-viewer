PDF.js 原理与项目集成说明

第一部分：PDF.js 提供什么

1. 整体架构

PDF.js 是 Mozilla 开发的开源 PDF 渲染引擎，完全用 JavaScript 实现。核心包 pdfjs-dist 包含三个文件：

主线程通过 postMessage 把解析任务交给 Worker，解析完成后把结果传回主线程渲染，不阻塞 UI。这是 PDF.js 性能优势的核心设计。

2. 渲染一页 PDF 的过程

PDF.js 把一页 PDF 渲染成两个独立的层，叠加在一起：

Canvas 层：把 PDF 的矢量指令转成像素，用户看到的所有视觉内容都来自这里。

Canvas 层：把 PDF 的矢量指令（画线、填色、绘字形）转成像素，用户看到的所有视觉内容都来自这里。文字是像素，不可交互。

TextLayer 层：用透明的 <span> 元素精确覆盖在 Canvas 上对应文字的位置，使文字可以被鼠标选中和复制。

3. API 调用顺序

getDocument(url)                    → PDFDocumentProxy
  └─ pdfDoc.getPage(n)              → PDFPageProxy
       ├─ page.getViewport({scale}) → PageViewport
       ├─ page.render({ctx,vp})     → RenderTask
       └─ page.getTextContent()     → TextContent

scale 控制缩放比例：1.0 = 72 DPI（PDF 原始分辨率），1.5 = 108 DPI。注意这个值越大渲染越慢。

4. TextItem：文字数据的基本单元

getTextContent() 返回一个 TextContent 对象，其中 items 是 TextItem[]——页面上所有文字片段的扁平列表，没有行、段落、列的概念，只有坐标。
