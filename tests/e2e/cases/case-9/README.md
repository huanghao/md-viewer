# Case 9：同步失败态停留与表单保持（纯失败）

## 目标
验证同步失败时：
- 弹窗不关闭；
- 失败状态块显示原始返回；
- 标题/父文档输入保持不变；
- 同步按钮恢复可点击（可继续重试）。

## 前置条件
- 已打开一个 Markdown 文件。
- 同步执行接口固定返回失败（测试桩）。

## 步骤
1. 打开同步弹窗并填写标题、父文档。
2. 点击同步（返回失败）。

## 预期结果
- 失败状态块可见，包含原始返回文本。
- 弹窗仍保持打开。
- 输入值与点击前一致。
- 同步按钮文案恢复为“同步”，且可再次点击。

## 失败信号
- 失败后弹窗关闭。
- 输入被重置。
- 按钮停留在“同步中...”或不可点击。
