# 20260305 评论系统改造交付稿（Draft）

文档定位：交付与实施草案（待确认）

## 1. 交付目标

在 `md-viewer` 内交付可用的本地评论系统：
1. 评论创建/回复/解决/重开。
2. 文中锚点与右侧评论面板双向定位。
3. 文档更新后的锚点自动重定位与异常兜底（orphan）。

## 2. 交付范围

### In Scope（v1）

1. 本地 SQLite 存储：文档、锚点、评论。
2. 评论面板：
- 筛选：全部/未解决/已解决/历史
- 列表模式：默认/极简
3. 评论状态流：`open -> resolved -> reopen`。
4. 锚点状态：`exact/reanchored/orphan`。
5. CLI 能力（沿用 `comments list/get` 风格）与 UI 数据一致。
6. 基础测试：单元 + 关键 E2E。

### Out of Scope（v1）

1. 批量翻译、跨文档聚合、复杂权限模型。
2. 云端同步。
3. 图片高级处理（压缩策略/外链抓取优化）。

## 3. 里程碑计划

1. M1 数据层（1 天）
- 建表与迁移
- repository 层与基本 CRUD
- 文档 hash 与版本记录

2. M2 锚点重定位（1.5 天）
- exact + context + fuzzy 三段匹配
- 置信度评分
- orphan 标记

3. M3 UI 与交互（1.5 天）
- 右侧面板与文中高亮联动
- 筛选/列表模式/历史区
- 解决/重开

4. M4 验证与发布（1 天）
- 单测 + E2E + 回归清单
- man 与 docs 同步

## 4. 验收标准

1. 功能验收
- 可创建评论并在刷新后保留。
- 解决后进入历史区，可重开回到未解决区。
- 点击评论卡/锚点均可正确定位。

2. 稳定性验收
- 文档小改动后，80% 以上评论可自动重定位到 `exact/reanchored`。
- 匹配失败评论不会错绑，必须标记 orphan。

3. 文档与命令验收
- `man/md-viewer.1`、`docs/配置说明.md`、`showHelp()` 同步。
- 命令输出适合 agent 上下文阅读。

## 5. 风险与对策

1. 风险：文档大改导致重定位命中率低。
- 对策：保守策略，不错绑；统一 orphan 流程。

2. 风险：评论渲染性能下降（长文 + 多评论）。
- 对策：评论列表虚拟滚动与锚点懒计算。

3. 风险：UI 状态复杂导致回归缺陷。
- 对策：状态机化与关键路径 E2E。
