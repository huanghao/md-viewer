# 输入新文件 - 文件输入界面重设计

## 待决策选项

### 1. 输入框位置
**选项 A：侧边栏中部独立区域（推荐）**
```
┌─────────────────────────┐
│ 🔍 搜索文件...          │
│ 📄 /current/path        │
├─────────────────────────┤
│                         │
│  ➕ 添加文件            │  ← 独立的输入区域
│  [输入文件路径...]      │
│                         │
├─────────────────────────┤
│ 📄 README.md            │
│ 📄 guide.md             │
└─────────────────────────┘
```

**选项 B：搜索框下方**
```
┌─────────────────────────┐
│ 🔍 搜索文件...          │
│ ➕ [输入文件路径...]    │  ← 紧贴搜索框
│ 📄 /current/path        │
├─────────────────────────┤
│ 📄 README.md            │
│ 📄 guide.md             │
└─────────────────────────┘
```

**选项 C：页面中央浮动输入框**
```
┌─────────────────────────────────────┐
│                                     │
│         ┌─────────────────────┐    │
│         │ 📂 添加 Markdown 文件│    │
│         │                     │    │
│         │ [输入文件路径...]   │    │
│         │                     │    │
│         │ 或拖拽文件到此处     │    │
│         └─────────────────────┘    │
│                                     │
└─────────────────────────────────────┘
```
- 触发方式：点击侧边栏底部的"➕"按钮，或快捷键 Cmd/Ctrl+O
- 显示在页面中央，类似对话框
- 点击外部或 Esc 关闭

**选项 D：保持当前位置，优化样式**
```
┌─────────────────────────┐
│ MD Viewer               │
│ 🔍 搜索文件...          │
│ 📄 /current/path        │
│ [输入路径...] [添加]    │  ← 保持在顶部
├─────────────────────────┤
│ 📄 README.md            │
└─────────────────────────┘
```

**推荐：选项 A**
理由：
1. 逻辑清晰：搜索和添加是两个不同的功能，应该分开
2. 视觉层次：搜索在顶部（高频），添加在中部（中频），文件列表在下方
3. 不干扰：不会与搜索框混淆
4. 易扩展：未来可以添加更多输入相关的功能（如文件浏览器）

### 2. 输入交互方式
**选项 A：单行输入框 + 回车添加（推荐）**
- 输入路径后按回车立即添加
- 按 Esc 清空输入
- 输入时显示路径提示（如果实现了路径补全）

**选项 B：输入框 + 添加按钮**
- 保留"添加"按钮
- 支持回车和点击按钮两种方式

**选项 C：多行输入框批量添加**
- 支持一次输入多个文件路径（每行一个）
- 按 Cmd/Ctrl+Enter 批量添加

**推荐：选项 A**
理由：
1. 最简洁，符合用户"输入路径"的需求描述
2. 回车添加是最自然的交互
3. 减少鼠标操作，提高效率

## 当前问题分析

### 现状
当前的文件输入在侧边栏顶部（`sidebar-header` 内）：
```html
<div class="sidebar-header">
  <h1>MD Viewer</h1>
  <div id="searchBox"></div>        <!-- 搜索框 -->
  <div id="currentPath"></div>      <!-- 当前路径 -->
  <div class="add-file">            <!-- 添加文件 -->
    <input type="text" id="fileInput" placeholder="输入文件路径...">
    <button onclick="addFile()">添加</button>
  </div>
</div>
```

### 问题
1. **功能混杂**：搜索、路径显示、添加文件都挤在顶部
2. **视觉拥挤**：顶部区域功能太多，层次不清晰
3. **交互不明确**：搜索框和添加文件输入框容易混淆
4. **空间浪费**：添加文件是中频操作，不需要一直占据顶部空间

## 测试场景

1. **输入路径添加**
   - 输入绝对路径 → 回车 → 文件添加成功
   - 输入相对路径 → 回车 → 文件添加成功
   - 输入不存在的路径 → 回车 → 显示错误提示

2. **拖拽添加**
   - 拖拽单个文件 → 文件添加成功
   - 拖拽多个文件 → 所有文件添加成功
   - 拖拽非 Markdown 文件 → 被忽略

3. **焦点管理**
   - 添加成功后 → 焦点状态符合预期
   - 添加失败后 → 焦点保持在输入框

4. **视觉效果**
   - 输入框聚焦 → 显示蓝色边框
   - 输入框失焦 → 恢复默认样式
   - 拖拽提示 → 清晰可见
