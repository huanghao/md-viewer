# Case 10：file-changed 交互（当前自动刷新 + 非当前 M + 点击 M 自动更新）

## 目标
验证文件变化事件在当前实现下的三条核心行为：
- 当前文件变化后自动更新正文（无需手动刷新）。
- 非当前文件变化后显示 `M` 状态。
- 点击带 `M` 的文件后自动读取新内容，并清除 `M`。

## 前置条件
- 启动本地服务并建立 SSE 连接。
- 使用本地临时 Markdown 文件 A/B。

## 步骤
1. 打开 A 和 B，保持 B 为当前文件。
2. 修改 B 内容。
3. 修改 A 内容。
4. 点击带 `M` 的 A。

## 预期结果
- 步骤 2 后，正文自动出现 B 的新内容。
- 步骤 3 后，A 在侧边栏显示 `M`。
- 步骤 4 后，正文出现 A 的新内容，且 A 的 `M` 消失。

## 失败信号
- 当前文件修改后仍停留旧正文，必须手动刷新才更新。
- 非当前文件修改后不显示 `M`。
- 点击 `M` 文件后正文不更新或 `M` 不消失。
