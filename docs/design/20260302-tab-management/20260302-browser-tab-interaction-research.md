# 浏览器 Tab 管理交互调研（针对 md-viewer）

文档定位（调研）：
- 提取主流浏览器在“多 tab 管理”上的交互模式。
- 为 md-viewer 的 tab 管理器提供可复用交互原则。

## 调研对象

1. Chrome（Tab Search + Tab Group + Close Others）
2. Safari（Tab Overview + Tab Group）
3. Arc（垂直 tab + 空间分组 + 已关闭恢复）

## 可复用模式

### 1) 搜索先于分组

- 大量 tab 场景下，用户更常先搜索而非先展开层级。
- 结论：md-viewer 管理器应将搜索框放在顶部，默认聚焦。

### 2) 批量动作显式可见

- 浏览器通常把 `Close Others / Close Tabs to Right` 放在上下文菜单。
- 结论：md-viewer 应把这类动作前置到面板顶部，减少右键记忆负担。

### 3) 最近使用优先

- Tab Search 结果往往按最近使用加权。
- 结论：默认排序采用 MRU，字母序作为补充。

### 4) “危险动作”轻确认

- 浏览器一般不弹重 modal，但会让动作语义明确。
- 结论：md-viewer 对 `关闭全部` 做轻确认文案（按钮二次点击/短确认区），其余动作直接执行。

## 不建议直接照搬

1. 深层级分组树：对 md-viewer 过重，维护成本高。
2. 跨窗口 tab 拖拽：非核心场景。
3. 复杂预览卡片：会遮挡正文并增加视觉噪声。

## 对 md-viewer 的建议映射

1. 顶部增加 `Tabs` 管理按钮。
2. 面板内固定四个批量动作。
3. 列表默认“最近使用 + 其他”两段，其他默认折叠。
4. 保持非模态，不引入新页面。
