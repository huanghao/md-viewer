# 侧边栏样式设计

## 待决策选项

### 1. 搜索框实现方式
**选项 A：实时过滤**
- 输入时立即过滤文件列表
- 高亮匹配的文本
- 支持模糊搜索（拼音首字母、部分匹配）

**选项 B：简单匹配**
- 只支持精确的子串匹配
- 不高亮，只显示/隐藏
- 实现简单，性能好

**选项 C：增强搜索**
- 支持正则表达式
- 支持按路径、文件名分别搜索
- 支持搜索历史记录

**推荐**：选项 A（实时过滤 + 高亮）

---

### 2. 当前文档路径的交互方式
**选项 A：点击复制整个路径**
- 点击路径文本直接复制到剪贴板
- 显示 toast 提示"已复制"
- 简单直接

**选项 B：显示复制按钮**
- 路径右侧显示复制图标
- 点击图标复制路径
- 更明确的交互提示

**选项 C：右键菜单**
- 右键路径显示菜单
- 支持"复制路径"、"在文件管理器中打开"等操作
- 功能更丰富

**推荐**：选项 B（显示复制按钮），因为更符合常见的 UI 模式

## 设计说明

### 参考图片分析
根据 `docs/leftSidebar.jpg`，设计要点包括：

1. **搜索框位置**：顶部固定
2. **当前打开文件样式**：透明的外围椭圆形边框
3. **普通文件背景**：正常背景，不使用特殊的蓝色背景
4. **当前文档完整地址**：可点击复制路径

### 功能细节

#### 1. 搜索功能
- 输入时实时过滤文件列表
- 匹配文件名（不区分大小写）
- 高亮匹配的文本部分
- 空搜索时显示所有文件
- 搜索框右侧显示清除按钮（有内容时）

#### 2. 路径复制功能
- 显示当前打开文件的完整绝对路径
- 路径文本可能很长，需要处理溢出：
  - 选项 A：截断中间部分，显示 `.../parent/file.md`
  - 选项 B：滚动显示（悬停时）
  - 选项 C：单行显示，overflow-x: auto
- 复制按钮：
  - 默认隐藏或半透明
  - 悬停时完全显示
  - 点击后调用 `navigator.clipboard.writeText()`
  - 显示 toast 提示"已复制路径"

#### 3. 文件列表样式
- 移除当前的蓝色背景（`.file-item.current`）
- 改为透明椭圆边框
- 普通文件保持简洁样式
- 悬停时显示浅灰色背景

### 技术考虑

#### 搜索性能
- 文件数 < 100：直接遍历过滤，性能足够
- 文件数 > 100：考虑 debounce（300ms）
- 使用 `String.prototype.toLowerCase()` 进行不区分大小写匹配

#### 路径处理
- 使用 `path.basename()` 获取文件名
- 使用 `path.dirname()` 获取目录路径
- 考虑 Windows 和 Unix 路径差异

#### 剪贴板 API
```typescript
async function copyToClipboard(text: string): Promise<boolean> {
  try {
    await navigator.clipboard.writeText(text);
    return true;
  } catch (err) {
    console.error('复制失败:', err);
    return false;
  }
}
```

### 可访问性

- 搜索框添加 `aria-label="搜索文件"`
- 复制按钮添加 `aria-label="复制文件路径"`
- 键盘快捷键：
  - `Ctrl/Cmd + F` 聚焦搜索框
  - `Esc` 清除搜索
  - `Ctrl/Cmd + Shift + C` 复制当前文件路径
