## 第六节：长段落（测试滚动条）

这一节包含较多内容，用于让文档超过一页，从而触发自定义滚动条显示。

段落一：软件开发是一门需要持续学习的技艺。技术在不断演进，新的工具和方法层出不穷。保持好奇心和学习的热情，是每个开发者最重要的品质。

段落二：代码审查是提升代码质量的重要手段。通过审查，团队成员可以互相学习，发现潜在问题，并确保代码符合项目规范。

段落三：测试是软件质量的保障。单元测试、集成测试、端到端测试各有其用途，合理组合才能构建可靠的测试体系。

段落四：文档是代码的重要组成部分。好的文档可以帮助新成员快速上手，也可以帮助未来的自己理解当初的设计决策。

段落五：性能优化需要基于数据。在没有性能问题之前不要过早优化，在有问题之后要先测量再优化，避免凭感觉优化。

段落六：安全性应该从设计阶段就开始考虑。输入验证、权限控制、数据加密，这些都是构建安全系统的基础。

段落七：可维护性比聪明的代码更重要。清晰、简单、一致的代码，比炫技式的复杂代码更有价值。

## 建议的修改方式

**测试新增行**：在某个列表末尾加一项，或新增一个段落。

**测试删除行**：删除某个列表项，或删除某段文字。

**测试修改行**：把"苹果"改成"芒果"，或修改代码块里的函数名。

**测试相邻变更**：同时修改第三节的两个代码块，观察它们是否合并为一个 diff 块。
