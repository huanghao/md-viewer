# 状态管理策略

日期: 2026-02-27

## localStorage 存储

```javascript
{
  files: [[path, { path, name, active, isRemote }]],
  currentFile: path
}
```

| 项目 | 说明 |
|------|------|
| 保存时机 | 打开文件、切换文件、关闭标签、删除文件 |
| 清除时机 | **从不主动清除**（除用户手动删浏览器数据）|
| 不存内容 | 只存元数据，内容每次刷新重新加载 |

**为什么只存元数据：** 文件可能被外部修改，刷新时重新加载保证内容最新。

## 状态矩阵

| # | 客户端有文件名 | 客户端有内容 | 服务端存在 | 场景 | 处理 |
|---|---------------|-------------|-----------|------|------|
| 1 | ✓ | ✓ | ✓ | 正常 | 显示 |
| 2 | ✓ | ✗ | ✓ | 刷新后 | 重新加载显示 |
| 3 | ✓ | ✓ | ✗ | 外部删除 | **问题：显示旧内容** |
| 4 | ✓ | ✗ | ✗ | 刷新后文件被删 | **问题：localStorage 残留** |
| 5 | ✗ | ✓ | ✓ | 不可能 |
| 6 | ✗ | ✗ | ✓ | 未打开 | - |
| 7 | ✗ | ✓ | ✗ | 不可能 |
| 8 | ✗ | ✗ | ✗ | 全新会话 | - |

## 发现问题

### 问题 1: localStorage 只增不减

用户点 X 删除文件会清理，但**文件被外部删除不会清理 localStorage**。

```
restoreState: 文件不存在 → 跳过加入 state.files
           ↓
         不调用 saveState()
           ↓
     localStorage 里永远残留
```

**影响：** localStorage 随时间膨胀，每次启动都尝试加载已不存在的文件。

### 问题 2: 场景 3 显示旧内容

外部删除文件后刷新，restoreState 加载失败但页面仍显示上次内存中的内容（如果有）。实际上如果之前没访问过这个文件，state.files 是空的，会显示空白。

## 删除行为

| 操作 | 函数 | 效果 |
|------|------|------|
| 左侧列表 × | `removeFile()` | 从 state.files 删除 + saveState |
| 右侧标签 × | `closeFile()` | 设为 inactive + saveState |
