# Case 6：父文档 URL 归一化与失焦元信息回填

## 目标
验证同步表单在输入父文档 URL 时：
- 命令预览中的 `--parent-id` 使用 URL 最后一段 ID；
- 输入框失焦后会拉取父文档元信息并展示标题链接。

## 前置条件
- 已打开一个 Markdown 文件。
- 同步相关 API 走测试桩（status/recent/preferences/parent-meta）。

## 步骤
1. 打开同步弹窗。
2. 在父文档输入框填写 URL（如 `https://xuecheng.com/doc/2748?from=e2e`）。
3. 触发输入框失焦。

## 预期结果
- 命令预览包含：`--parent-id "2748"`。
- 父文档元信息区域显示标题，并且是可点击链接。

## 失败信号
- 命令预览仍使用完整 URL（未提取 ID）。
- 失焦后不展示父文档标题或展示空白。
