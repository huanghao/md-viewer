3. 索引服务设计

RAG server 是一个独立的 Bun 进程，与主 md-viewer server 分离...

3.1 架构概览

RAG server 启动后加载 embedding 模型，初始化 SQLite 表，读 rag_meta 对比模型名。

文件系统事件（chokidar）
启动 chokidar watch 所有工作区目录。
文件新增/修改 → 防抖 2s 后入队，文件删除 → 立即删除对应 chunks + vectors。

重索引队列串行执行，单次一个文件：读文件 → 分块 → batch encode → 事务写库。

3.2 索引调度策略

触发时机：RAG server 启动时增量扫描，文件保存时防抖 2s 后入队，文件删除立即处理，模型变更时全量重索引。
