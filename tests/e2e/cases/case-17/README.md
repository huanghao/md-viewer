# Case 17：删除后同路径重建并恢复监听

## 目标
验证已打开文件删除后进入 `D`，同路径重建后可恢复为正常可读，并继续接收后续变更（进入可刷新状态）。

## 前置条件
- 简单模式。
- 本地 Markdown 文件 A 可被打开（`tests/e2e/runtime/case-17/e2e-delete-recreate-a.md`）。

## 步骤
1. 创建并打开 A。
2. 删除 A，等待删除态 `D`。
3. 同路径重建 A（新内容）。
4. 重新打开 A 触发恢复。
5. 再次修改 A，观察是否继续进入可刷新状态并可刷新成功。

## 预期结果
- 删除后显示 `D` 与内容区删除提示。
- 重建并重新打开后，`D` 清除并显示新内容。
- 后续再修改 A，可继续出现 `M`/刷新按钮并刷新成功（监听恢复）。

## 失败信号
- 重建后仍长期停留在 `D`。
- 重新打开无法恢复正文。
- 恢复后再修改不再出现可刷新状态，或刷新后内容不更新。
